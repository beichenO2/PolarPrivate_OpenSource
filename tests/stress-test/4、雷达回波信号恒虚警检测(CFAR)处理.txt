      5.1 实验背景与核心意义
    恒虚警检测（Constant False Alarm Rate, CFAR）是雷达目标检测的核心技术，其核心价值在于：通过目标邻近的参考单元自适应估计背景杂波/噪声的平均功率，动态生成与背景水平匹配的检测阈值，在预设的虚警概率下实现目标的稳健检测

三类经典CFAR算法，覆盖不同探测场景：page21
    1. 单元平均CFAR（CA-CFAR）：本实验基础教学核心算法，适用于均匀背景环境。通过对CUT两侧所有参考单元的功率取算术平均，完成背景功率估计，算法实现简单、计算量小，适配室内回环测试与低杂波室外场景。
 2. 单元平均选大CFAR（GO-CFAR）：适用于杂波边缘场景（如目标靠近建筑、地面强反射区）。分别计算CUT左右两侧参考单元的平均功率，取两者中的较大值作为背景功率估计值，可有效抑制杂波边缘区域的虚警率飙升问题，适配室外复杂边界场景的目标检测。
3. 单元平均选小CFAR（SO-CFAR）：适用于多目标密集场景。分别计算CUT左右两侧参考单元的平均功率，取两者中的较小值作为背景功率估计值，可避免邻近强目标导致的背景功率高估，减少多目标场景下的弱目标漏检，适配本实验多目标测距、测角的极限分辨率测试场景。

5.4.2 2D距离-多普勒谱CFAR处理流程（适配测速、测角实验）page24
该流程用于距离-多普勒二维域目标检测，适配本实验运动目标测速、多目标测角场景，核心是将1D滑窗扩展为2D滑窗，步骤如下：
1. 输入数据预处理：输入多普勒FFT后的2D距离-多普勒谱（RDM），对天线维度做非相干积累，得到距离-多普勒二维功率谱矩阵；
2. 2D CFAR参数配置：距离维、多普勒维分别配置保护单元与参考单元，距离维参数同1D CFAR，多普勒维保护单元推荐2、参考单元推荐48，预设虚警概率与1D保持一致；
3. 二维滑窗遍历检测：以二维滑窗遍历RDM所有距离-多普勒单元，对每个CUT，在二维参考单元内完成背景功率估计，生成二维自适应阈值；
4. 目标检测与匹配：筛选出超过阈值的有效检测单元，完成距离门与多普勒门的匹配，建立目标距离与速度的一一映射关系；
      5. 测角流程衔接：提取有效目标对应的距离门、多普勒门数据，完成MIMO虚拟阵列拼接与角度FFT，实现目标角度的稳健估计。

补充：目标检测“点迹凝聚处理”
雷达波束扫过目标时，因目标尺寸大于分辨率单元或过采样，同一目标会在相邻距离 / 多普勒单元产生多个过门限点迹；将这些同源离散点合并为一个代表目标真实位置的中心点迹的处理过程，是连接信号检测与航迹跟踪的关键环节。
工程中最简单的点迹凝聚处理：
9 点法（九宫格 / 3×3 邻域凝聚）与25 点法（二十五宫格 / 5×5 邻域凝聚），是雷达点迹凝聚中基于固定邻域窗的幅度加权质心算法，本质是在距离 - 方位二维平面上，以当前峰值点为中心，取周围 3×3 或 5×5 窗口内的过门限点迹，合并为一个中心点迹，工程上称为邻域窗凝聚法。

（六）雷达回波信号恒虚警检测（CFAR）处理 page72
1.基础概念题：相比于传统的固定阈值检测，CFAR 恒虚警检测是雷达实际工程应用中的核心目标检测方法，请回答：① CFAR 检测的核心优势是什么？其解决了固定阈值检测的哪些核心痛点？② 解释本实验采用的 CA-CFAR 算法的核心原理，并分别说明保护单元与参考单元的作用。
2.原理分析题：在雷达检测理论中，虚警概率与检测概率存在固有的折中关系。请结合本实验的 2D CA-CFAR 代码回答：① 当增大相对门限 dB 偏移量时，系统的虚警概率与检测概率会分别发生什么变化？② 当减小相对门限 dB 偏移量时，系统的虚警概率与检测概率会分别发生什么变化？③ 在本实验的行人步行 / 跑步测速场景中，应如何平衡二者，选择合适的门限参数？
3.工程拓展题：本实验代码采用的 CA-CFAR 算法，在杂波边缘场景（如目标靠近建筑、地面强反射区）中，容易出现虚警率飙升的问题。请说明该问题的产生原因，并将代码中的 CA-CFAR 算法替换为 GO-CFAR 算法，写出核心的修改步骤与阈值计算公式的变化。
